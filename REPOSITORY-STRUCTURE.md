## File Naming Conventions

### General Rules
- Use lowercase with hyphens for file names: `ai-tools-overview.pptx`
- Include version numbers when appropriate: `syllabus-v2.0.pdf`
- Date stamp documents: `lecture-notes-2025-01-15.pdf`
- Use descriptive names: `chapter-03-exercise-curriculum-mapping.pdf`

### Specific Conventions
- Presentations: `##-descriptive-name.pptx`
- Exercises: `exercise-##-topic.pdf`
- Assessments: `assessment-type-topic.pdf`
- Figures: `fig-##-descriptive-name.ext`
- Code: `descriptive-function-name.py` or `.ipynb`

## Content Standards

### Each Chapter Must Include:

1. **README.md**
   - Chapter overview
   - Learning objectives
   - Key concepts
   - Estimated teaching time
   - Prerequisites
   - Navigation to resources

2. **Presentations** (3-5 per chapter)
   - PowerPoint format (.pptx)
   - Consistent University of Melbourne branding
   - Speaker notes included
   - Editable format

3. **Lecture Notes**
   - Instructor guide (PDF)
   - Detailed lecture notes (Markdown or PDF)
   - Teaching tips and common challenges
   - Discussion prompts

4. **Exercises** (3-5 per chapter)
   - Clear instructions (PDF)
   - Solution guides
   - Supporting datasets
   - Estimated completion time

5. **Assessments**
   - Multiple assessment types
   - Detailed rubrics
   - Marking guides
   - Sample solutions/exemplars

6. **Figures**
   - High-resolution images (PNG/SVG)
   - Properly attributed
   - Source files when applicable
   - Caption list with citations

7. **Readings**
   - Required and supplementary lists
   - Bibliography in BibTeX format
   - Links to accessible versions
   - Reading guides

8. **Code/Software** (where applicable)
   - Well-commented code
   - Requirements.txt or environment files
   - Setup instructions
   - Usage examples

9. **Resources**
   - Supplementary materials
   - External links
   - Tool guides
   - Additional references

## Quality Standards

### Presentation Files
- University of Melbourne branded template
- Consistent fonts and colors
- High-quality images (min 300 DPI)
- Accessible design (contrast, alt text)
- Speaker notes on every slide

### Exercise Files
- Clear learning objectives
- Step-by-step instructions
- Expected outputs defined
- Time estimates provided
- Difficulty level indicated

### Assessment Materials
- Aligned with learning outcomes
- Detailed rubrics (4-6 criteria)
- Clear submission requirements
- Academic integrity guidelines
- Accessibility considerations

### Code Files
- PEP 8 compliant (Python)
- Comprehensive docstrings
- Unit tests where applicable
- Requirements file
- Usage documentation

### Figure Files
- Vector formats preferred (SVG, EPS)
- Minimum 300 DPI for raster images
- Proper attribution and citations
- Accessible color schemes
- Consistent styling
