Inside your chapter folder, organize files as follows:

* **Data/Spreadsheets:** Upload to the `/datasets` subfolder.
* **Python/Code:** Upload to the `/code` subfolder.

### 2. File Naming Conventions
To maintain order, please use the following naming pattern:
* `Ch01_Draft_v1.docx` (for Word documents)
* `Ch01_Figure_1-1.png` (for images)
* `Ch01_Script_Analysis.py` (for code)

### 3. How to Submit Changes
We recommend the following workflow for this project:

#### Option A: Web Interface (Easiest for non-coders)
1.  Navigate to your specific chapter folder on GitHub.
2.  Click **"Add file"** â†’ **"Upload files"**.
3.  Drag and drop your documents.
4.  In the "Commit changes" box, write a brief description (e.g., "Uploaded first draft of Chapter 5").
5.  Select **"Create a new branch for this commit and start a pull request"**.
6.  Click **"Propose changes"**.

#### Option B: Git/Desktop App
1.  Clone the repository.
2.  Create a new branch: `git checkout -b feature/chapter-name`.
3.  Add your files locally.
4.  Commit and Push: `git push origin feature/chapter-name`.
5.  Open a Pull Request on GitHub.
